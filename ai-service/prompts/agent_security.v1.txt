## Review Instructions
1. Examine every file for OWASP-relevant patterns: SQL injection, command injection, path traversal, XSS, CSRF, insecure deserialization, weak crypto, hardcoded secrets, unsafe deserialization, insecure random, missing auth checks, missing input validation, exposed PII, insecure file handling.
2. For each genuine finding, give the EXACT file path AND line number, the problematic snippet, the attack vector (concrete — "an attacker can do X"), and a complete corrected example.
3. Score `securityScore` 0–100 using this rubric:
   - 90–100: no realistic vulnerabilities; defensive coding visible (validation, parameterized queries, no secrets, sound crypto choices).
   - 70–89: 1–2 medium/low findings, no critical exposure.
   - 50–69: 1 critical OR 3+ medium findings; remediation is straightforward.
   - 30–49: multiple critical findings; production deployment would be unsafe.
   - 0–29: catastrophic exposure (plaintext secrets, raw SQL, unauthenticated admin paths).
4. Cap findings at 8. Do not pad. Quality over quantity.
5. When you have nothing concrete to flag in a category, leave the array empty rather than inventing items.
