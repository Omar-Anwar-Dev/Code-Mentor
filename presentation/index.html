📝 Feature 2: Adaptive Assessment

التقييم التكيّفي — كيف نقيس مستوى المتعلم بدقة

مكونات السؤال

  • Question Text — نص السؤال
  • 4 Choices — اختيارات MCQ
  • Category — واحدة من 5 فئات
  • Difficulty (b) — صعوبة السؤال (−3 → +3)
  • Discrimination (a) — تمييز السؤال
  • Explanation — شرح الإجابة

كيف تُنشأ الأسئلة؟

1️⃣ AI يولّد الأسئلة + يقدّر (a,b)
↓
2️⃣ Admin يراجع ويعدّل
↓
3️⃣ Data تُعايِر (1000+ إجابة)

📝 IRT — كيف يختار السؤال الأمثل

P(θ) = 1 / (1 + e−a(θ−b))
المعاملالمعنىالمدى
θ (theta)قدرة المتعلم−4 → +4
bصعوبة السؤال−3 → +3
aتمييز السؤال0.3 → 3.0

Fisher Information — يختار السؤال الذي يوفر أكبر قدر من المعلومات عن مستوى المتعلم الحالي

θ₀ = 0.0 (Start)
↓
Select Q → max Fisher Info
↓
Student Answers ✓ / ✗
↓
Update θ via MLE (scipy)
↓
Repeat × 30 Questions

🛤️ Feature 3: Learning Path

مسار تعلم مخصص يتكيّف مع مستوى كل متعلم

Stage 1: Embedding Recall

ملف المتعلم (θ scores + weaknesses) → text-embedding-3-small → Cosine Similarity مع كل المهام → أقرب 20 مهمة

Stage 2: LLM Rerank

20 candidate + سياق المتعلم → GPT-5.1-codex-mini → مسار مرتب من 5-10 مهام مع prerequisites

Schema ✓ Real IDs ✓ Topology ✓ Retry on fail ✓

🎯 Token Management + Prompt Engineering

إدارة الـ Tokens

  • Budget tracking — كل endpoint يُسجّل tokens
  • Reasoning effort: low — يركز output لا thinking
  • Context window — نقص الملفات الأهم فقط
  • $50/month soft cap مع per-learner tracking

Prompt Engineering

  • Versioned prompts — كل prompt في ملف منفصل
  • Template + variables — context injection
  • Self-correction retry — يعالج أخطاءه
  • JSON repair — strip fences + balanced block

لماذا مهم؟

Prompt design يؤثر مباشرة على جودة المراجعة — prompt سيئ = نتائج سطحية
